# Subagents

## 핵심 요약

서브에이전트는 메인 Claude 안에서 **별도 컨텍스트**를 가진 워커. 자기만의 시스템 프롬프트 · 허용 도구 · 권한을 가지고 독립 실행된다. 핵심 가치: **(1) 병렬 처리**, **(2) 컨텍스트 보호** (파일 100개 뒤져도 결과만 메인에 돌아옴), **(3) 전문화**, **(4) 재사용**. 토큰 총량 관리를 위해 **Sonnet/Haiku 권장, Opus 는 메인에만**.

## 서브에이전트 vs 메인 대화

| | 메인 대화 | 서브에이전트 |
|---|---|---|
| 컨텍스트 | 공유 (모든 파일 탐색 결과 누적) | 격리 |
| 병렬성 | 순차 | 동시 실행 가능 |
| 통신 | 사용자와 직접 | 메인에게 요약만 반환 |
| 적합 | 순차적 작업, 컨텍스트 공유 필요 | 독립적 작업, 오염 우려 |

> **판단 기준**
> 1. 작업이 독립적인가? → 독립적이면 서브에이전트
> 2. 컨텍스트 오염 우려? → 우려되면 서브에이전트

## 4대 핵심 패턴

### 격리 (Isolation)

별도 컨텍스트에서 작업 수행 후 요약만 메인에 반환. 예: 테스트 실행 → 결과만 돌아옴, 파일 내용은 메인 오염 없음.

### 병렬 (Parallel)

독립적 작업 동시 실행. **"10분 걸릴 작업을 3분에"**.

예: 백엔드 구현 / 프론트엔드 구현 / 테스트 작성 → 3개 서브에이전트 동시 실행.

### 체인 (Chain)

한 에이전트 결과를 다음 에이전트로 전달:

```
분석 에이전트 → 리팩터 에이전트 → 테스트 에이전트 → 리뷰 에이전트
```

### 재개 (Resume)

중단된 작업을 이어서 실행.

## 내장 서브에이전트

Claude Code 가 내부적으로 활용하는 서브에이전트들:

| 에이전트 | 역할 | 모델 |
|---|---|---|
| **Explore** | 코드 탐색, 파일 검색, 구조 파악 | Haiku (빠르고 저렴) |
| **Plan** | 계획 수립 | — |
| **General Purpose** | 다단계 작업 | — |
| **Bash** | 별도 세션에서 터미널 명령 실행 | — |
| **Claude Code Guide** | Claude Code 기능 질문 답변 | — |

**Explore 가 Haiku 를 쓰는 게 핵심**. 코드베이스 탐색은 비용이 커서 Opus 로 하면 토큰이 금방 증발.

## 모델 선택 원칙

> 서브에이전트에 Opus 쓰지 말라.

**이유**: 서브에이전트는 보통 병렬/다량 실행된다. 10개 에이전트가 동시에 Opus 를 쓰면 토큰이 순식간에 소진. 토큰은 총알이다.

- ✅ **Sonnet** — 기본값
- ✅ **Haiku** — 탐색/검색 특화
- ❌ **Opus** — 메인에만

## 커스텀 에이전트 생성

```
/agents
```

선택:
1. Create new agent
2. 프로젝트 레벨 vs 개인 레벨
3. Generate with Claude (추천) → 시스템 프롬프트 자동 작성
4. Tool 선택 (읽기 전용 / 편집 / 실행 / MCP / 전체)
5. 모델 선택 (Sonnet 기본)
6. 색상 (구분용, 예: 빨강 = 테스트, 파랑 = 백엔드)
7. Agent Memory enable (셀프러닝 로컬 캐시)

### 저장 위치

| 위치 | 사용 범위 |
|---|---|
| `.claude/agents/` (프로젝트) | 해당 프로젝트, git 공유 가능 |
| `~/.claude/agents/` (개인) | 모든 프로젝트 |
| **CLI 일회성** | 세션 한정 (컨텍스트 효율적) |

## 안티 패턴

- ❌ **의존적인 작업을 무리하게 병렬화** — 서로 결과를 기다려야 하면 병렬 의미 없음
- ❌ **하나의 서브에이전트에 역할 몰빵** — 테스트 + 마이그레이션 + 프론트 + 백엔드 전부? 전문화 상실
- ❌ **결과 검증 없이 그대로 사용** — 반환값은 반드시 메인에서 검증
- ❌ **서브에이전트끼리 직접 통신 시도** — 메인이 코디네이터 역할. (Agent Teams 가 이를 개선)

## Agent Teams (신기능)

서브에이전트끼리 **직접 메시지 교환** 가능. 실제 개발팀처럼:

- 프론트 담당
- 백엔드 담당
- 테스트 담당

각자 독립 컨텍스트를 유지하면서 필요할 때 상호 통신. 리더 에이전트가 작업 조율.

> 저자가 "에이전틱 엔지니어링의 진짜 중요한 기능" 이라고 평가.

## Skills · MCP · Subagents 비교

| | 안 쓸 때 | 실행 중 |
|---|---|---|
| **Skills** | 설명만 (50~100B) — 가장 가벼움 | on-demand 로드 |
| **MCP** | 도구 설명 전부 상시 | 호출 시 추가 |
| **Subagents** | 정의 상시 (MCP 와 비슷) | **별도 컨텍스트** — 메인 오염 없음 |

결론: **안 쓸 때 = Skills 승, 실행 중 = Subagents 승**. 상황에 맞게 조합.
