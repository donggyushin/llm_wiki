## 핵심 요약

새 직장에서 *AI 코딩 노하우 공유* 가 명시적 기대치로 걸려 있다. 4개 질문에 대한 본인 답을 정리한 노트. 핵심 4축: **(1) 도구 스택**, **(2) 코드베이스 구조화 원칙**, **(3) AI 비즈니스 임팩트 사례**, **(4) AI 에게 시키지 말아야 할 일**. 4번이 가장 시니어 시그널 — *"컨텍스트 깊이"* 를 기준으로 잡고 레거시 / 조직 정치 / Production incident 3종을 AI 에서 제외.

## Q1. AI 코딩 도구 스택

| 작업 종류 | 사용 도구 |
|---|---|
| 터미널 코딩 | Claude Code |
| 문서 정리 | Obsidian + Claude Code 조합 |

> [!note] 시사점
> Cursor / Copilot / Windsurf 가 아닌 **CLI 기반 Claude Code** 를 메인으로 선택. IDE 통합형보다 **세션·컨텍스트·hook 제어권** 을 우선. Obsidian + Claude Code 조합은 이 위키 자체와 동일 패턴 → [[concepts/llm-wiki-pattern]].

## Q2. AI 가 잘 돌도록 코드베이스를 어떻게 구조화하나

5대 원칙:

1. **모듈별 CLAUDE.md 분할** — root `CLAUDE.md` 는 최대한 간소. 다른 문서들을 참조하는 허브 역할만. → [[concepts/claude-md]]
2. **markdown writer agent + hook 강제 동기화** — 코드 변경 시마다 문서 동기화를 의무화. hook 으로 강제. → [[concepts/hooks]]
3. **TDD hook 강제** — 테스트 코드도 hook 으로 작성 강제.
4. **`docs/` 안에 ADR 정리** — Architecture Decision Records 로 의사결정 사유 누적.
5. **MCP 다이어트** — 컨텍스트 최적화를 위해 무거운 MCP 자제. 쉽게 구축 가능한 MCP 면 직접 만들거나 script 파일로 대체. → [[concepts/context-engineering#3-mcp-다이어트]]

> [!note] 일관 철학
> 5개 원칙 모두 *"AI 가 규칙을 읽고도 어기는 문제"* 를 **시스템 강제** 로 풀려는 [[concepts/harness-engineering|Harness Engineering]] 사고. 프롬프트는 부탁, hook 은 강제.

## Q4. "AI 에게 시키지 말아야 할 일" — 시니어 시그널

**기준: 컨텍스트 깊이 — 암묵지 많은 작업은 AI 에게 시키지 않는다.**

코드만 봐서는 알 수 없는 정보가 결정에 필요한 영역에서 AI 는 약하다. 3종:

| 영역 | AI 가 약한 이유 |
|---|---|
| **레거시 수정** | "왜 이렇게 짰는지" 의 역사를 모름. *의도된 이상함* 을 버그로 오인하고 멀쩡한 코드를 "개선" → 6개월 전 incident 패치를 되돌릴 위험 |
| **조직 정치 얽힌 코드** | 팀 경계·소유권·책임 분담은 코드에 안 적혀 있음. AI 가 *기술적으로 깔끔한 안* 을 내도 PR 은 다른 팀 소유라 막힘 |
| **Production incident 근본원인 분석** | 로그 + 메트릭 + 사람 기억 + 배포 히스토리를 합쳐야 보임. AI 는 코드만 봄. *그럴듯한 답* 에 끌려가면 진짜 원인 놓침 |

**AI 활용 방식 — 가설 brainstorm 까지만. 결정은 사람.**

| 영역 | AI 활용 한계선 |
|---|---|
| 레거시 | "이 코드의 가능한 의도 5가지" 까지. 수정 여부는 git blame + 동료 확인 후 |
| 조직 정치 | "기술적으로 깔끔한 안" 까지. 정치적 실현 가능성은 사람 |
| Incident | 코드 가설까지. 메트릭/로그/배포 히스토리는 사람이 검증 |

→ 상세: [[concepts/what-not-to-delegate-to-ai]]

## 본인 입장의 일관성

4개 답이 한 줄로 묶임: **AI 의 신뢰 영역과 사람의 신뢰 영역을 명확히 분리한다.**

- Q1: 제어권 우선 (CLI 도구 선택)
- Q2: 시스템 강제 (hook 으로 규칙 박음)
- Q3: 병렬화·자동화 (AI 가 잘하는 영역에 더 부담)
- Q4: 컨텍스트 깊이가 부족한 영역은 AI 에서 제외

이 일관성 자체가 면접에서의 *시니어 시그널*.
